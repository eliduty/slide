# 工作区目录规范

每个演示的 `workspace/` 目录用于管理素材和工作文件，与最终产出分离。

## 目录结构

```
workspace/
├── research.md           # 调研笔记
├── outline.md            # 大纲草稿
├── references.md         # 参考资料
├── scripts/              # 演讲脚本
│   ├── cover.md
│   └── intro.md
├── data/                 # 数据源文件
├── drafts/               # 原始草稿文件
└── feedback.md           # 反馈记录
```

## 各目录用途

### research.md

调研笔记，记录知识搜集、来源链接、要点整理。

**格式示例**：

```markdown
# 调研笔记

## 来源

- [文章链接1](url) - 关键要点
- [文章链接2](url) - 关键要点

## 核心概念

- 概念1：说明
- 概念2：说明

## 待深入

- 问题1
- 问题2
```

### outline.md

演示结构规划，对应 slides/ 文件映射。

**格式示例**：

```markdown
# 大纲

## 1. 封面 (01-cover.md)
- 标题展示
- 作者介绍

## 2. 项目介绍 (02-intro.md)
- 项目背景
- 核心概念
- 应用场景

## 3. 总结 (03-summary.md)
- 回顾要点
- 下一步行动
```

### references.md

参考资料列表。

**格式示例**：

```markdown
# 参考资料

## 文章

- [标题](链接) - 简要说明

## 书籍

- 书名 - 作者 - 相关章节

## 视频

- [标题](链接) - 简要说明
```

### scripts/

演讲脚本（逐字稿），与 slides/ 文件对应。

**命名规则**：与 slides/ 文件名对应，去掉序号前缀。

- `slides/01-cover.md` → `scripts/cover.md`
- `slides/02-intro.md` → `scripts/intro.md`

**格式示例**：

```markdown
# 封面页脚本

时长：约 1 分钟

## 开场白

大家好，今天我要分享的是...

## 重点提示

- 强调标题关键词
- 简短自我介绍
```

### data/

图表、数据可视化所需的原始数据文件。

**常见格式**：

- `.csv` - 表格数据
- `.json` - 结构化数据
- `.xlsx` - Excel 文件

### drafts/

原始草稿文件，处理后导出到 public/。

**常见内容**：

- `.excalidraw` - Excalidraw 图表源文件
- `.drawio` - Draw.io 图表源文件
- `.png/.jpg` - 手绘草稿、未处理图片

### feedback.md

观众反馈记录。

**格式示例**：

```markdown
# 反馈记录

## 2026-03-20 演讲反馈

### 正面反馈

- 内容清晰易懂
- 案例很有帮助

### 改进建议

- [ ] 增加更多代码示例 - 处理中
- [x] 尝试速度稍微放慢 - 已处理

## 待处理事项

- 需要补充案例演示页面
```
