# AI Agent 核心概念解析

Rules、Agent、MCP、Skill、Memory、Command 都属于 AI Agent（智能体）体系的核心能力与组件，共同构成了让 AI 从"被动回答"升级为"自主完成任务"的智能体架构（Agentic AI）。

## 一、核心概念与作用

### 1. Agent（智能体）

AI 的自主主体，是整合所有能力的"执行者"。

- **定位**：具备感知、思考、决策、行动闭环的独立 AI 实体
- **能力**：理解目标 → 自主规划 → 调用工具/技能 → 执行任务 → 反馈调整
- **类比**：像一个"数字员工"或"智能助理"

### 2. Skill（技能）

Agent 的可执行能力库，即"工具箱"。

- **定位**：模块化、可插拔的任务执行单元
- **能力**：封装特定功能（搜索、读文件、写代码、发邮件、数据分析）
- **特点**：可组合、可调用、可复用
- **类比**：人的专业技能（会开车、会编程、会做饭）

### 3. Memory（记忆）

Agent 的信息存储与检索系统。

- **定位**：解决 AI"记不住"问题，实现有状态、个性化交互
- **分类**：
  - **短期记忆**：当前对话/任务上下文（会话级）
  - **长期记忆**：用户偏好、历史记录、经验知识（跨会话）
- **类比**：人的大脑记忆（短期工作记忆、长期经验记忆）

### 4. Rules（规则）

Agent 的行为约束与安全边界。

- **定位**：静态、明确、不可违背的指令与限制
- **作用**：定义合规逻辑、业务流程、安全红线、价值导向
- **例子**：
  - "不生成违法内容"
  - "价格低于成本时禁止下单"
- **类比**：人的道德准则、法律底线、公司制度

### 5. MCP（Model Context Protocol，模型上下文协议）

Agent 的中枢调度与通信框架。

- **定位**：管理上下文、记忆、技能、模型的核心控制层
- **核心能力**：
  - **上下文传递**：跨模型/跨系统共享记忆与状态
  - **技能注册与调用**：统一接口调用各类 Skill/工具
  - **会话管理**：维护多轮对话连贯性
  - **安全沙箱**：保障调用安全
- **类比**：人体的神经系统+大脑皮层，协调全身动作与信息流转

### 6. Command（指令）

Agent 执行操作的最小单元，即"动作指令"。

- **定位**：Skill 的具体执行载体，是 Agent 与外部系统交互的"命令语言"
- **能力**：封装具体操作参数、执行逻辑、返回格式
- **特点**：原子化、可序列化、可追踪、可撤销
- **例子**：
  - "读取文件 /path/to/file"
  - "执行 SQL 查询 SELECT * FROM users"
- **类比**：人的"动作指令"（抬手、迈步、按下按钮）

## 二、六者关系

**一句话总结**：

Agent 是"人"，MCP 是"大脑+神经"，Memory 是"记忆"，Skill 是"手脚/技能"，Rules 是"规矩"，Command 是"动作指令"。

### 详细关系

| 组件 | 角色 | 职责 |
|------|------|------|
| **Agent** | 统筹一切 | 设定目标、做决策 |
| **MCP** | 内部调度 | 负责协调、通信、上下文管理 |
| **Memory** | 信息支撑 | 提供历史信息与上下文支撑决策 |
| **Skill** | 执行能力 | 提供具体执行能力完成任务 |
| **Rules** | 行为约束 | 确保行为合规、安全、可控 |
| **Command** | 执行指令 | Skill 的具体执行单元，将意图转化为可执行的操作 |

---

## 三、一句话区分

| 组件 | 核心问题 | 说明 |
|------|----------|------|
| **Agent** | 谁来做 | 主体 |
| **MCP** | 怎么协调、怎么连 | 调度/通信 |
| **Skill** | 会做什么、怎么做 | 执行能力 |
| **Memory** | 记住什么、用什么历史 | 信息存储 |
| **Rules** | 不能做什么、必须遵守什么 | 约束 |
| **Command** | 具体做什么动作 | 执行指令 |
