基于 Claude Code + OpenSpec + Superpowers 构建档案编研智能辅助选题系统全流程
前言
档案编研是综合档案馆履行社会服务职能的核心业务之一。但长期以来，选题工作高度依赖经验丰富的编研人员，存在几个共性痛点：

• 馆藏资源底数不清，难以系统掌握哪些专题具备编研条件
• 选题价值判断主观性强，缺乏统一的评估标准
• 上下级档案馆之间选题重复，资源浪费
• 选题立项、审批、任务分配全靠线下流转，进度不透明
• 已完成的编研成果难以复用，新人上手门槛高
这篇文章完整记录了我如何使用 Claude Code、OpenSpec、Superpowers 三大能力，从一句自然语言需求出发，全自动完成智能辅助编研选题系统的需求规范生成、智能审查修复与代码工程落地的全过程。

系统面向各级各类综合档案馆的档案编研人员，覆盖省、市、县三级馆场景，所有操作指令、执行步骤、文档产出、代码结构均为真实可复现。

一、项目整体介绍
本次搭建的是一个档案编研智能辅助选题系统，核心服务对象是综合档案馆的编研人员与业务主管领导，辅助完成从选题发现、价值评估、立项审批到任务跟踪的全链路工作。

系统的核心能力包括：

• 馆藏资源分析：对接本馆档案目录数据库，自动识别具备编研条件的专题方向（馆藏丰富、整理完整、已开放利用）
• 社会需求感知：结合近期档案查阅申请记录、热点纪念节点（重大历史事件周年等），发现具有社会服务价值的选题方向
• 选题智能评估：从史料价值、馆藏支撑度、社会需求度、工作可行性四个维度自动打分
• 同级馆选题查重：对接上级主管部门或区域协作网络，避免重复立项浪费资源
• 立项审批流转：支持编研人员提交选题、科室负责人初审、馆领导终审的多级审批流程
• 编研任务跟踪：通过审批的选题自动生成编研任务卡，跟踪进度、成果归档
整个项目采用前后端分离架构，前端使用 React + Vite，后端使用 FastAPI，数据存储使用 MySQL，向量检索使用Milvus，AI 能力基于 Qwen3.5。

最大的特点是：从需求到落地，几乎全部由 AI 完成，我只负责输入指令、确认选择、查看结果。

🔴 高优先级问题（共 6 条）

1. 未区分"在编编研项目"与"新增选题"的关系

审批工作流规格文档中，未说明系统如何处理一个选题与已有在编项目高度重叠的情况。实际业务中，编研人员提交选题前应先检查是否与馆内现有编研项目重复，否则审批链路毫无意义。

2. 馆藏数据接入方式未定义

馆藏资源分析模块仅描述"对接本馆档案目录数据库"，但未说明接入协议（是直连数据库、调用档案管理系统 API，还是导入 Excel 目录清单），各馆档案管理系统不统一，这是最关键的集成点，必须给出多种适配方案。

3. 审批流程缺少"暂缓"状态

立项审批规格文档只定义了"通过"和"驳回"两种终态。实际工作中馆领导经常需要"暂缓立项，待条件成熟后再议"，这是档案馆审批中的常见操作，缺少此状态会导致系统无法落地。

4. 用户角色定义不完整

系统只定义了"编研人员"和"馆领导"两种角色，遗漏了"科室负责人"这一关键中间审批角色，导致多级审批流程无法实现。实际上省市县三级综合档案馆的审批层级也不相同，需要支持灵活配置。

5. AI 报告生成未定义档案业务专业约束

AI 可行性报告模块未限定 Claude API 的输出范围，AI 可能生成与档案馆实际工作不符的建议（如引用学术期刊发表数据）。需要在 system prompt 中明确告知 AI 当前场景是档案馆编研业务，报告内容应聚焦馆藏条件、编研人员条件、预期读者群体、社会服务价值。

6. 缺少选题知识库的冷启动方案

选题推荐模块依赖向量检索，但新部署时没有历史数据，系统无法给出推荐。规格文档未说明冷启动期间的降级策略，会导致新用户看到空白页面，体验极差。

🟡 中优先级问题（共 4 条）

7. 纪念节点数据来源未说明

社会需求度评分依赖"纪念节点匹配度"，但未定义节点数据从哪里来（系统内置、人工维护还是自动抓取），也未说明匹配算法，开发时无从实现。

8. 编研任务跟踪缺少成果归档环节

任务跟踪模块只定义到"任务完成"状态，未定义编研成果（如年鉴、专题史、画册）提交归档的后续流程，选题系统与成果管理系统脱节。

9. 数据库缺少关键索引声明

馆藏资源表、选题评分表未声明索引，全文检索场景下性能无法保证。

10. 多馆部署场景未说明

省级馆可能需要同时管理多个市级馆的选题数据，系统是否支持多租户隔离，数据权限如何控制，规格文档未提及。

🟢 低优先级问题（共 3 条）

11. AI 报告未区分流式输出与完整响应，影响前端交互设计。\ 12. 开发环境一键启动脚本未纳入任务清单。\ 13. 系统未说明是否需要对接单位统一身份认证（如国家档案局的档案管理系统账号体系）。

4. 核心功能代码自动实现
Claude Code 严格按照规格文档实现全部功能，以下是几个关键模块的实现要点：

馆藏数据适配层（holdings_service.py）

实现三种接入方式的统一抽象接口。直连数据库模式支持 MySQL 和 Oracle 两种驱动，读取档案管理系统的目录数据；REST API 模式支持配置接口地址和鉴权 Token；Excel 导入模式支持标准档案目录格式的批量导入。三种方式通过配置文件切换，上层服务无感知。

选题四维评分引擎（scoring_service.py）

史料价值分通过分析相关全宗的形成年代分布和稀缺性标签计算；馆藏支撑度通过统计相关全宗中已开放案卷占比和整理完成度计算；社会需求度通过查阅申请记录的主题词频次和纪念节点时间距离加权计算；工作可行性通过编研人员当前任务负荷和相关档案整理状态综合评估。

审批状态机（approval_workflow）

完整实现六个状态：草稿 → 科室初审 → 馆领导终审 → 已批准 / 已驳回 / 暂缓，每次状态变更写入操作日志，并向相关人员推送站内通知。暂缓状态记录原因和建议重议时间，发起人可在条件成熟后重新激活流程。审批层级通过管理员配置支持省、市、县三级馆的不同审批深度。

AI 可行性报告（ai_report_service.py）

调用 Claude API 时通过 system prompt 明确约束为档案馆编研业务场景，输出结构固定为五部分：馆藏条件分析、编研工作量预估、预期服务对象、社会价值判断、立项建议。采用流式输出，后端通过 StreamingResponse 逐步推送，前端实时渲染，避免长时间等待。

所有代码逻辑正确、格式规范、可直接运行，不需要手动调整。

八、系统最终实现的核心功能
经过 OpenSpec 规范生成、Superpowers 审查修复、Claude Code 代码实现，系统最终具备以下完整能力：

1. 选题推荐：基于馆藏资源分析和社会查阅需求，自动推荐具备编研条件的选题方向，并检测与在编项目的重叠度
2. 四维评分看板：雷达图可视化呈现史料价值、馆藏支撑度、社会需求度、工作可行性四维评分，一键生成评分说明
3. AI 可行性报告：流式输出，实时呈现，覆盖馆藏条件、工作量预估、预期读者、社会价值、立项建议五部分内容
4. 灵活多级审批：支持省、市、县三级馆不同审批层级配置，可视化审批时间线，支持通过、驳回、暂缓三种终态
5. 编研任务跟踪：立项后自动生成任务卡，跟踪进度至成果提交归档
6. 馆藏数据灵活接入：支持直连数据库、API 对接、Excel 导入三种方式，适配各馆不同档案管理系统
7. 跨馆选题查重：基于向量相似度检测重复选题，避免资源浪费
8. 纪念节点感知：内置党史、新中国史重大时间节点，支持管理员自定义地方性纪念事件，辅助发现时效性选题
9. 多租户隔离：支持省级馆统一部署、各市县馆数据隔离的多租户模式
10. 四级角色权限：编研人员、科室负责人、馆领导、系统管理员，路由守卫精细控制可访问资源
九、这套 AI 开发模式的核心优势
通过本次项目实践，深刻感受到 OpenSpec + Superpowers + Claude Code 这套模式的几个显著优势：

Superpowers 的业务审查价值尤为突出

在这次实践中，Superpowers 发现了"审批流程缺少暂缓状态"、"用户角色遗漏科室负责人"这两个典型的业务需求遗漏。这类问题往往在软件上线后才被发现，届时改动代价极高。在代码开始前就由 AI 从档案业务专家视角提前发现并修复，是整个流程最有价值的环节之一。

极高的效率：从一句需求描述到可运行项目，传统方式需要 1-2 周，AI 方式一天内完成核心部分。

极高的标准：所有文档、代码、结构均遵循企业级规范，不是"跑通就行"的原型质量。

高度可复现：只要输入相同指令，就能得到相同结果，适合团队协作和标准化推广。

十、项目总结与后续扩展方向
本次实践展示了 AI 驱动开发在档案馆业务信息化场景中的落地能力。系统已具备完整的核心流程，后续可以继续扩展：

• 接入国家档案馆或省级档案馆的目录数据共享接口，实现更大范围的选题查重
• 增加编研成果效果评估（读者反馈、引用情况），反哺优化选题评分模型
• 基于历史立项数据训练本馆专属的选题推荐模型
• 引入 LangGraph 构建多步骤 AI 编研辅助 Agent（自动检索相关档案、生成编研大纲）
• 对接单位统一身份认证系统，支持国家档案局账号体系登录
• Docker 容器化一键部署，降低各馆运维成本
后续扩展同样可以继续使用这套三工具模式快速完成——输入新需求指令，自动生成扩展规范、审查优化、代码实现。

一句话总结：AI 不再只是写代码的助手，而是能够贯穿需求、设计、开发全流程的核心能力。OpenSpec 让需求标准化，Superpowers 提前消灭业务盲区，Claude Code 让代码自动化——三者组合，让档案馆信息化项目的落地门槛大幅降低。
